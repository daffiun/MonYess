# UI Direction - FinanceBot (Refined)

Dokumen ini merinci arah visual final untuk FinanceBot, menggabungkan gaya **Modern Fintech** dengan elemen **Gamifikasi Ringan**.

## 🎨 Visual Direction: Modern Fintech
Aplikasi akan menggunakan gaya SaaS modern yang bersih, memberikan kesan profesional, tenang, praktis, dan terpercaya.

- **Background**: Putih bersih (`#FFFFFF`) untuk area utama, abu-abu sangat muda (`#F8FAFC`) untuk latar belakang halaman.
- **Surface**: Kartu putih dengan border tipis berwarna Slate (`#E2E8F0`) dan *soft shadow*.
- **Sidebar (Desktop)**: Warna netral Slate/Zinc (`#0F172A`) untuk memberikan kontras yang elegan.

## 🌈 Color System
- **Primary**: **Blue** (`#2563EB`) - Digunakan untuk aksi utama, tombol CTA, dan elemen navigasi aktif.
- **Secondary/Neutral**: **Slate/Gray** (`#475569`) - Digunakan untuk teks sekunder, border, dan elemen UI netral.
- **Success/Income**: **Green** (`#10B981`) - Digunakan untuk pemasukan, arus kas positif, dan status sukses.
- **Danger/Expense**: **Red** (`#EF4444`) - Digunakan untuk pengeluaran, arus kas negatif, dan status peringatan/error.
- **Warning/Accent**: **Amber/Orange** (`#F59E0B`) - Digunakan untuk indikator budget yang hampir habis atau reminder hutang.

## 🔠 Typography Direction
- **Font Family**: **Plus Jakarta Sans** (atau Inter sebagai fallback).
- **Style**: Tipografi bersih dengan hierarki yang jelas. Judul menggunakan *bold*, teks isi menggunakan *regular/medium*.

## 🧩 Component Style
- **Rounded Corners**: Sudut membulat sedang (misal: `rounded-2xl` untuk kartu, `rounded-xl` untuk tombol).
- **Borders**: Garis tipis 1px yang halus, tidak mencolok.
- **Shadows**: *Subtle shadows* untuk memberikan kedalaman tanpa terlihat berat.

## 📱 Layout Structure
- **Desktop**: Sidebar tetap di kiri (warna Slate), Header di atas, Konten di tengah (fluid grid).
- **Mobile**: Bottom Navigation untuk akses jempol yang mudah. Header minimalis. Masukan transaksi cepat diutamakan.

## 🎮 Gamification Style
- **Approach**: Halus dan profesional. Menggunakan ikon kecil, progress bar yang tenang, dan kartu informasi yang compact.
- **Tone**: Mendukung dan netral. Tidak menghakimi kondisi finansial user.

## 🚫 What to Avoid
- Visual yang terlalu "kekanak-kanakan" atau penuh warna-warni berlebihan.
- Animasi yang mengganggu atau memperlambat workflow.
- Elemen gamifikasi yang bersifat menghakimi atau mempermalukan user.
- Penggunaan leaderboard publik (privasi adalah prioritas).
